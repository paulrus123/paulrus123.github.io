Cloud Anchors Beer Pong

A shared experience using Google's Cloud Anchors

Github Link

Try it out!

Background

This project is a follow up of the MagicPutt shared AR experience. In MagicPutt, world localization is achieved through a fiducial (tracked image). In this project I used Google's CloudAnchor technology, which can localize a physical spaced based on a cloud hosted "anchor". I host a cloud anchor from a mobile device, and can access that same anchor on a second phone. Then I use a similar MQTT messaging strategy to create a multiplayer user experience.

Interactions

I tried to put thought into making an easily understandable AR experience - especially with setting up the play area. The user has to be able to understand how to track a surface, place an augmented object via an anchor, or resolve the anchor on a different device. Once the table is placed, also making easily understandable gameplay mechanics was considered.

Syncing experience

This clip shows an example of how a cloud anchor is hosted and resolved. Each hosted cloud anchor uses a generated ID that other devices can resolve. Synchronization is occured using MQTT for fast messaging. This is configurable to use a public broker or for better performance a locally hosted broker. Future plans include to replace this with a hosted server that can connect multiple clients via a local connection over WiFi for best performance. Even with the slowest (public broker) messaging option, I was able to use a few tricks to make the ball movement seem fluid on both the host and client devices.

Multiplayer

Part of this project tried to get "real-time" feeling multiplayer interactions. The goal (still in progress!) is for the defending player to be able to "block" the ball after the player shoots it.



Dev Notes

Improvements from MagicPutt include transparent shadows to reflect actual ground space, follower behavior to improve sync between devices, and using cloud anchor points instead of fiducials for localization.
