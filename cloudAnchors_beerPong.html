Cloud Anchors Beer Pong

A shared experience using Google's Cloud Anchors

Github Link

Try it out!

Background

This project is a follow up of the MagicPutt shared AR experience. In MagicPutt, world localization is achieved through a fiducial (tracked image). In this project I wanted to try Google's CloudAnchor technology, which can localize a physical spaced based on a cloud hosted "anchor". I host a cloud anchor from a mobile device, and can access that same anchor on a second phone. Then I use a similar MQTT messaging strategy to create a multiplayer user experience.

Syncing experience

Please see the video for an example of how a cloud anchor is hosted and resolved. Each hosted cloud anchor uses a generated ID that other devices can resolve. Synchronization is occured using MQTT for fast messaging. This is configurable to use a public broker or for better performance a locally hosted broker. Future plans include to replace this with a hosted server that can connect multiple clients via a local connection over WiFi for best performance. Even with the slowest (public broker) messaging option, I was able to use a few tricks to make the ball movement seem fluid on both the host and client devices.

Dev Notes

Improvements from MagicPutt include transparent shadows to reflect actual ground space, follower behavior to improve sync between devices, and using cloud anchor points instead of fiducials for localization.
