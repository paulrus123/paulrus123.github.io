Augmented Reality Sensor Visualization

Zenuity / Personal

Unity developer for augmented reality sensor visualizations

Autonomous and ADAS enabled vehicles in development and on the road today have a host of sensors that help them understand and navigate the world around them. Engineers (including myself!) create tools to visualize this data. Usually it's in a 2D or 3D rendering format you can view on a computer. However I wanted to be able to see actual visualization around the car in a 1:1 scale.

Streaming Data to a mobile device for augmented reality viewing

This first project was an attempt to stream point cloud data to a mobile device and visualize it in AR. A point cloud is generated from a RADAR or LIDAR sensor. Complex perception algorithms (which I don't develop!) can understand this point cloud and be able to start classifying roads, vehicles, pedestrians, and other objects. As you can see, I don't have a car with these types of sensors, so I am sending over a "random" point cloud. Github Link

Zenuity Hackathon - Zensor Leap

As a follow-up to the above project, I formed a hackathon team with a few coworkers to further develop this concept using a MagicLeap one headset. We tried to first visualize sensor configurations using preset properties from actual sensors. Although we weren't able to finish the sensor visualization we made a lot of progress and had some fun too. Check out our video of the results! Github Link
